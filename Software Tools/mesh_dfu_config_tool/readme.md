# Zephyr BLE Mesh LPN DFU Configuration Tool

Zephyr BLE Mesh LPN DFU Configuration Tool is a web-based utility for generating optimal Kconfig values for Zephyr BLE Mesh Device Firmware Update (DFU) deployments. It configures three node roles (Distributor, Friend, LPN) and provides real-time LPN DFU duration estimate based on firmware size.

Note: This tool is designed for Zephyr BLE Mesh implementations. Configuration values are generated in Kconfig format for easy copy-paste into your project.

## Prerequisites

- Modern web browser (Chrome, Firefox, Edge, Safari) with JavaScript enabled
- All three files (`index.html`, `styles.css`, `script.js`) must be in the same directory

## Usage

Simply open `index.html` in your web browser:

```bash
# Linux
xdg-open index.html

# macOS
open index.html

# Windows
start index.html
```

### Configuring Your Mesh Network

1. **Distributor Node**: Configure TX/RX segment parameters and message counts
2. **Friend Node**: Configure TX/RX segment parameters, message counts, queue size, and chunk request parameters
3. **LPN Target**: Configure TX/RX segment parameters, message counts, and BLOB report timeout
4. **Firmware Size**: Enter your LPN application firmware size

The tool automatically:
- Calculates negotiated chunk sizes and segment counts
- Validates Friend queue capacity for your configuration
- Provides real-time recommendations for optimal settings
- Estimates DFU duration for 1 LPN target in unicast mode

### Copying Configuration

Click the **"Copy Kconfig Options to Clipboard"** button to copy your configuration values to the clipboard. The button will show a confirmation message when successful.

Paste the copied configuration into your Zephyr project's `prj.conf` file:

```
# BLE Mesh DFU Configuration
# Generated by BLE Mesh DFU Configuration Tool

# ========================================
# DISTRIBUTOR Configuration
# ========================================
CONFIG_BT_MESH_TX_SEG_MAX=10
CONFIG_BT_MESH_RX_SEG_MAX=10
CONFIG_BT_MESH_TX_SEG_MSG_COUNT=1
CONFIG_BT_MESH_RX_SEG_MSG_COUNT=5
CONFIG_BT_MESH_BLOB_CLI=y
CONFIG_BT_MESH_DFU_CLI=y
CONFIG_BT_MESH_BLOB_BLOCK_SIZE_MIN=1024

# ========================================
# FRIEND NODE Configuration
# ========================================
CONFIG_BT_MESH_FRIEND=y
CONFIG_BT_MESH_FRIEND_QUEUE_SIZE=46
CONFIG_BT_MESH_FRIEND_RECV_WIN=255
CONFIG_BT_MESH_TX_SEG_MAX=10
CONFIG_BT_MESH_RX_SEG_MAX=10
CONFIG_BT_MESH_TX_SEG_MSG_COUNT=1
CONFIG_BT_MESH_RX_SEG_MSG_COUNT=2
CONFIG_BT_MESH_BLOB_BLOCK_SIZE_MIN=1024

# ========================================
# LPN TARGET Configuration
# ========================================
CONFIG_BT_MESH_LOW_POWER=y
CONFIG_BT_MESH_RX_SEG_MAX=10
CONFIG_BT_MESH_TX_SEG_MAX=10
CONFIG_BT_MESH_TX_SEG_MSG_COUNT=1
CONFIG_BT_MESH_RX_SEG_MSG_COUNT=1
CONFIG_BT_MESH_BLOB_SRV=y
CONFIG_BT_MESH_BLOB_SRV_PULL_REQ_COUNT=4
CONFIG_BT_MESH_DFU_SRV=y
CONFIG_BT_MESH_BLOB_REPORT_TIMEOUT=10
CONFIG_BT_MESH_LPN_RECV_DELAY=100
CONFIG_BT_MESH_LPN_POLL_TIMEOUT=300
```

**Note:** The FRIEND_RECV_WIN, LPN_RECV_DELAY, and LPN_POLL_TIMEOUT values shown above are Zephyr defaults. During active DFU operations, Zephyr caps the poll timeout at 1 second regardless of the configured value, so these parameters have minimal impact on DFU duration.

For more information on any Kconfig option, visit https://docs.zephyrproject.org/3.7.0/kconfig.html
